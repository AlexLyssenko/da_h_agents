## 2. App-Level Admin Role (`User.isAdmin`)

**Requirement:** Frontend Task 9.1 — "show only if user is app-level admin — add `isAdmin` flag to User model"

**What exists:**
- `AdminPage.tsx` renders correctly but has no role guard
- `isAdmin` exists only on `RoomMember` (room-level admin), NOT on `User`

**What is missing:**
- `isAdmin Boolean @default(false)` column on the `User` model
- A new Prisma migration
- The backend login/refresh responses must include this flag in the `user` object
- `authStore` must expose `user.isAdmin`
- `AdminPage` and its `/admin` route must be gated behind `user.isAdmin === true`

**Files to edit:**
```
backend/prisma/schema.prisma          ← add isAdmin field to User
backend/src/services/auth.ts          ← include isAdmin in login/refresh responses
frontend/src/App.tsx                  ← guard /admin route
frontend/src/store/authStore.ts       ← add isAdmin to User type
```

**Command after schema edit:**
```bash
cd backend && npx prisma migrate dev --name add-user-isadmin
```
